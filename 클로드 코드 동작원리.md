# 애이전틱 루프 
- 컨텍스트 수집 -> 행동실행 ->결과 검증 3단계 루프
- 컨텍스트 수집
	- 프로젝트 파일탐색
	- 관련코드 읽기
	- git 상태확인
	- 에러 메서지 분석
		-> 파일 읽고 코드 구조 파악 및 현재 상황 이해
- 행동실행
	- 파일 수정
	- 명령어 실행
	- 테스트 실행
	- git 커밋
		-> 수집 정보 바탕으로 작업 수행
- 결과 검증
	- 테스트 통과 여부 확인
	- 코드 문법 오류검사
	- 예상결과 실제 결과 비교
		-> 수행 작업 올바른지 확인

# 세션과 컨텍스트
## 1. 핵심 개념 (Core Concepts)

### 1.1 세션 (Session)

- **정의:** 클로드 코드와의 대화 및 작업 히스토리를 담은 영구적인 기록 단위.
    
- **저장 위치:** 로컬 파일 시스템의 `~/.claude/sessions/` 경로에 JSON 형태로 저장됨.
    
- **특징:**
    
    - 터미널 종료나 재부팅 후에도 유지됨 (**Persistence**).
        
    - 프로젝트별로 여러 세션을 가질 수 있으며, 작업 흐름을 분리하여 관리 가능.
        

### 1.2 컨텍스트 (Context)

- **정의:** 현재 추론을 위해 모델이 실시간으로 참조하고 있는 '머릿속' 정보.
    
- **구성 요소:** 최근 대화 기록, 현재 읽어온 파일 내용, 터미널 명령 결과값 등.
    
- **제한 사항:** '컨텍스트 윈도우(토큰 제한)'가 존재하며, 너무 길어지면 성능 저하나 비용 상승이 발생함.

## 2. 세션 제어 명령어 (Command Reference)

|**명령어**|**단축어**|**설명**|
|---|---|---|
|`claude`|-|새로운 세션을 시작 (기존 맥락 없음)|
|`claude --continue`|`claude -c`|현재 디렉토리의 **마지막 세션**을 즉시 재개|
|`claude --resume`|`claude -r`|저장된 **세션 목록**을 보고 선택하여 재개|
|`claude --list-sessions`|-|모든 저장된 세션의 ID와 생성 시간 확인|
|`exit`|-|현재 세션을 저장하고 종료|

---
### Session Fork (세션 포크)

- **개념:** 기존 작업 흐름의 '체크포인트'를 생성하여 새로운 독립 세션으로 분리하는 기법.
    
- **사용 시점:**
    
    - 현재 코드를 유지한 채 위험한 리팩토링을 시도할 때.
        
    - 하나의 프로젝트에서 여러 가지 하위 기능을 동시에 개발할 때.
        
    - 컨텍스트가 너무 무거워져서 필요한 부분만 떼어내어 가볍게 시작하고 싶을 때.
        
- **장점:** > * **안전성:** 원본 작업 내역 보존.
    
    - **효율성:** 컨텍스트 오염 방지 및 토큰 최적화.
        
    - **유연성:** 다양한 시나리오를 동시에 테스트 가능

## 3. 컨텍스트 최적화 기법 (Context Management)

### 3.1 토큰 관리 및 압축 (`compact`)

대화가 길어지면 컨텍스트가 가득 차서 모델이 예전 내용을 잊기 시작합니다.

- **`claude compact`**: 현재까지의 긴 대화 내용을 요약(Summary)하여 컨텍스트 공간을 확보합니다.
    
- **효과:** 비용 절감 및 모델의 집중도 향상.
    

### 3.2 파일 수동 관리

클로드 코드는 에이전트로서 파일을 스스로 읽고 쓰지만, 사용자가 직접 제어할 수도 있습니다.

- 특정 파일을 컨텍스트에서 제외하고 싶을 때: `.claudeignore` 파일에 경로 추가.
    
- 불필요한 파일이 너무 많이 읽혔을 때: 세션을 새로 열거나(New Session) 수동으로 정리를 지시.
    

---

## 4. 세션 유지 관리 (Maintenance)

### 4.1 세션 삭제

불필요하게 쌓인 세션은 디스크 용량을 차지하므로 정리가 필요합니다.

- **명령어:** `claude --delete-session <SESSION_ID>`
    
- **전체 삭제:** `rm -rf ~/.claude/sessions/*` (로컬 데이터이므로 물리적 삭제 가능)
    

### 4.2 세션 활용 전략

1. **Feature-based Session:** 새로운 기능을 개발하거나 복잡한 버그를 잡을 때는 새 세션을 열어 컨텍스트를 깨끗하게 유지.
    
2. **Context-Rich Session:** 프로젝트 전반의 아키텍처를 논의하거나 지속적인 가이드가 필요할 때는 `-c` 옵션으로 맥락을 유지.
